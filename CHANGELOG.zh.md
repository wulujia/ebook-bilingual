# 更新日志

[English](CHANGELOG.md) | **简体中文**

## 0.3.2

- **`extract` 不再因 EPUB 每个文档内嵌 HTML 注释而崩溃。** 基于内容的前置/后置件检测器
  (`looks_like_matter`) 会扫描 `root.iter()` 找第一个标题,但 `root.iter()` 也会吐出注释节点和处理
  指令节点,它们的 `.tag` 是个可调用对象而不是字符串。kobo/Calibre 导出的 EPUB 会在每个章节文件里塞
  一个 `<!-- kobo-style -->` 注释,于是去名字空间用的 `el.tag.split("}")` 撞上这个可调用对象,抛出
  `AttributeError`,在任何翻译开始前就让整个运行中止(在 qntm 的《There Is No Antimemetics
  Division》上触发)。标题扫描现在会跳过非元素节点(`isinstance(el.tag, str)`),与该文件其余裸
  `root.iter()` 循环本就容忍注释的写法保持一致。检测行为没有其他变化——保留了不依赖名字空间的标题
  匹配。`TestMatterDetection` 新增一个带前导注释节点的用例。

## 0.3.1

- **重建出来的目录不再以作者*另一本*书的标题开头。** 当一本书没有可用的目录页时,navMap 会改用每个
  spine 文档自身的标题来重建(即"A"兜底)。"Also By the Author"(本书作者的其他著作)卡片列的是作者的
  其他书名——而 Calibre 把这种页面排成纯粹一摞斜体书名,页面上根本没有"Also By"字样——于是标题提取取了
  第一个书名,把它当成了第一个 navPoint:Isaacson 的《Benjamin Franklin》打开后第一条竟是
  "Kissinger: A Biography"。兜底逻辑现在能识别"作者其他著作"卡片(没有标题、两行及以上斜体书名、没有
  正文散文),并把它排除在 navMap 之外。献词("For Ina""and for Dr. Travell")同样是几行短斜体,所以
  检测器会拒绝那些以献词式开头(For / To / and for…)为主的页面,从而保住真正的前置件。已在 28 本书
  (3341 个文档)上验证:只命中那两张真正的"Also By"卡片,别无其他。只改动 `toc.ncx`,且仅在没有目录页
  的兜底路径下生效。`TestTocRebuild` 新增覆盖该卡片、它的不同标记写法,以及必须放过的献词/章节页。

## 0.3.0

- **EPUB 译完后,章节终于能在阅读器目录里跳转了** —— 翻译 EPUB 时只把中文注入正文文件,源书的
  `toc.ncx` 原封不动,而 Z-Library 扫描版常带一个残缺的 `<navMap>`(只有 Front Cover / Copyright /
  Back Cover,别无其他),于是译好的双语书打开后,导航目录一个章节都跳不到。`repackage` 现在会重建
  navMap。**B(优先)**:解析书自带的目录页——它的 `<a>` 链接给出准确的标题和正确的目标。
  **A(兜底)**:没有目录页时,用每个 spine 文档自身的标题(首个 `<h1..6>`,或对那些 Z-Library 把
  标题排成段落的章节,取一个简短、含字母的首个 `<p>`),并跳过没有标题的续页碎片。两者在阅读顺序的
  spine 上合并,所以 playOrder 正确、B 的标题优先,而 A 仍能补回目录页错链或漏掉的章节——在
  The Power Broker 上,目录页把"49. The Last Stand"错链到了上一个文件(Chap48),只有遍历 spine 才能
  把第 49 章在正确位置补回来。目录页绝不会把自己列进去;一旦 B 定好了目录,A 只补*编号*章节,所以
  献词、图版插页、后页附录不会被拖进来。The Power Broker 实测:从 6 条、零个可跳章节的残缺目录 →
  54 条干净条目(导言 + 全部 50 章 + 书自己目录里列出的图版/注释页),没有空标题、没有死链。只改
  `toc.ncx`——`pageList`、正文、译文逐字节不动。尽力而为:书坏了也照样能打包。新增 `TestTocRebuild`
  测试覆盖标题提取、目录页解析、B/A 合并、navMap 重写。

## 0.2.7

- **QA 批次计数错乱不再让其中的段落永远漏过二次复核** —— L2 语义回查把段落按每批 8 个发给评审，要求每对
  恰好一条 verdict；评审偶尔会把两对合并或拆开、返回 N±1 条，于是抛出 `qa length mismatch`，`cmd_qa` 把
  *整批* 丢弃，8 段全部停在 `l1flag`。又因为重跑会把同一批 flagged 行以同样顺序重新分批，同一个"毒"段落
  反复弄坏同一批，被卡住的行永远收敛不到 0（在《通往权力之路》上实测：一本 40.8 万词的书在反复 `qa` 之后
  仍有 16 段永远没判定，每次都报 `qa length mismatch: 7 vs 8`）。真正的风险是静默的：藏在这种批次里的真
  误译会跳过 L2 及其 L3 修复，未经复核就进了成品。新增的 `qa_judge_robust()` 在计数不匹配时二分——与
  `translate_robust()` 对称——这样一个毒段落连累不了同批的邻居；实在判不出的孤段落给一条保守的
  `faithful=0` verdict，把它送去 L3 修复而不是静默放行。超时仍照常向上传播，下次运行重试。已由新增的
  `TestQABatchResilience` 测试覆盖。

## 0.2.6

- **worker 失败时点明真正原因，而不是打印一长串命令** —— `claude -p` 调用失败时抛出的
  `TimeoutExpired`/`CalledProcessError`，其字符串*开头*就是完整的命令 repr（里面嵌着几千字符的系统
  提示词），所以打印到控制台、写进 `units.error` 列的 `str(e)[:300]` 永远只是那段命令转储——超时和
  CLI 报错根本分不出来；在一次真实运行里（Steve Jobs EPUB）约 6% 的失败 unit 因此无从诊断。新的
  `concise_error()` helper 现在把它们渲染成 `timed out after 240s` 或 `exited 2: <stderr 尾部>`
  （保留消息的**结尾**，真正的原因在那里），`translate` 和 `qa` 都改用它。附带效果：这顺手修好了
  `translate` 的限流退避——原来 `rate.?limit|overloaded|429` 检测是在命令转储上匹配，从来匹配不到，
  指数退避一直形同虚设。

- **含"抱歉/sorry"的忠实译文不再被从双语成品里丢弃** —— `_META` 垃圾过滤器（负责把 AI 拒答/元话术挡在
  注入之外）此前用裸 `抱歉` 匹配，于是任何译文里在正常对白中出现该词的段落（"'这么早打扰您真是抱歉,'那名
  记者说"）会被悄悄只注入英文。在 Code Breaker EPUB 上这丢掉了 2 段完全正确、且通过 QA 的译文。现在 `抱歉`
  只有在其后几字之内跟着拒答动词（`无法`/`不能`/`不会`/`没办法`）时才命中，真正的拒答仍抓得住，被引用的
  对白则保留下来。已由新增的 `TestMetaFilter` 测试覆盖。

- **输出重定向时进度逐行实时可见** —— `main()` 现在把 `sys.stdout` 设为行缓冲，于是
  `python3 ebook_bilingual.py run … > run.log`（或任何非 TTY 重定向）会在每条 `✓`/进度行产生时
  立即刷出。CPython 对重定向后的 stdout 是块缓冲，所以此前后台运行的日志在进程退出前一直是空的，
  `tail -f run.log` 根本看不到进度，真实进展只能查 `cache.sqlite`。已用
  `hasattr(sys.stdout, "reconfigure")` 兜住非常规流。

- **不带书名的 `status` 现在是多 run 总览面板** —— 裸 `status` 此前会从 `active.txt`（一个谁后写谁赢的
  指针，任何并行 run 都会覆盖它）解析出单本，于是多书会话里常常报错书。现在它列出 `runs/` 下每个 run 的
  unit/段落进度和 QA 失败数，用 `→` 标出上次活动的那本；`status --book <slug>` 仍可下钻单本。只读
  （`mode=ro`），绝不阻塞某个进行中 run 正在写的 DB。纯格式化函数 `fmt_run_line()` 由 `TestStatusDashboard`
  覆盖。

- **QA 改为增量式——对已完成的书重跑不再花费任何 claude 调用** —— `qa`（以及 `run` 的 `qa` 步）此前会对
  *每一条* 译文重跑 L1，把各自的 `qa_state` 重置回 `l1ok`/`l1flag`，再重新抽样、用 `claude -p` 重判全书
  20% 的段落。于是对已完成的书重跑 `run`（比如只想重建 EPUB）会悄悄把整个 L2 判定重花一遍。现在它只判定
  刚被（重新）翻译的行：`translate` 每次写入都会把 `qa_state` 重置为 `untested`，QA 正好只处理这些（外加
  某次中断运行残留、需续判的 `l1flag` 行），已判定的一律跳过。已完成的书重跑即空操作；只重译了一部分则
  只重判改动的部分；首次运行行为不变。新增纯函数 `qa_worklist()` 由 `TestQAIncremental` 覆盖。（只想重建
  EPUB 时，`repackage` 仍是最直接的路径。）

## 0.2.5

- **抽取失败时响亮中止,而不是生成空书** —— 当发现阶段找不到可翻译段落(全部为空、低于
  `--min-words`、被 `--skip` 命中,或被识别为前后页废料)时,`extract` 现在会带着点明各个开关的诊断
  中止(退出码 1),而不再打印 `✓ extract done … 0` 后继续产出一本空的双语 EPUB。这种"假成功"正是
  0.2.3 的 `index_split` bug 一直隐形的原因。`discover_targets` 里的解析失败现在也会报出来(哪个
  文件、什么错误),不再被静默跳过。

## 0.2.4

- **`run --pdf` 像 `run --epub` 一样触发重抽取**：给 `run` 传源文件现在两种格式都会（重新）抽取；
  此前只有 `--epub` 会，于是 `run --pdf book.pdf` 会悄悄沿用旧的 units。重抽取是幂等的（同 PDF →
  同段落哈希 → 全部命中缓存），续跑依然很便宜。
- **回退到上次活动书时给出警告**：不带 `--book`/`--epub`/`--pdf` 的命令会用 `runs/active.txt` 里的
  slug，而并发运行会互相改写它；现在会打印一行警告，避免多书会话里悄悄操作错书。要明确就传 `--book`。

## 0.2.3

- **Calibre / Z-Library 导出的 EPUB 不再抽出零章** —— 这类工具把整本书拆成名为
  `index_split_<n>.html` 的逐章文件,而默认 `--skip` 列表里的 `index` token 以子串方式命中了其中
  每一个,于是整本书被过滤掉(`0 content docs`)。移除了 `index` 这个文件名 token;真正的索引页仍由
  基于内容的前后页检测(`looks_like_matter`)拦截。一本典型的 Z-Library 长篇(《The Mountain in
  the Sea》)现在能抽出 54 章(其中 1 个真索引页按内容自动跳过),而不再是 0 章。

## 0.2.2

- **页码变化也能剥掉的页眉**：页眉以"短语 + 页码"形式重复出现，页码每页都变，所以旧的"整行完全相同"
  频率检测从来没认出它；在那些因边栏而走 `-raw` 抽取的书里，它漏进了约六分之一的段落，有时还插进词
  中间（`the teleLinus Torvalds and David Diamond xix phone`）。现在计数前先把页码归一化，并在页面
  任意位置剥除页眉行（不只页眉页脚），顺带让跨页断开的词重新接上（`tele-`/`phone` → `telephone`）。
  页码按规范罗马数字校验，所以由罗马字母组成的英文词（`did`、`mill`）不会被误当页码。
- **重抽取时清理孤儿译文**：重抽取改变段落文本后（例如清洗修复），旧文本的译文不再滞留缓存；只保留被
  当前段落引用的行，QA 和报告不再处理永远不会出现在成品里的文本。

## 0.2.1

- **修复：日期密集的短章节不再被当成索引跳过**：基于内容的前后页废料检测里，有一条会把"几乎全是
  短条目、且大多带页码样数字"的页面判定为索引/目录。一篇*短*的叙事章节，若塞满笔记本引文、日期和
  脚注编号（如 *Leonardo da Vinci* 第 19 章「Personal Turmoil」，约 1700 词），就会有 60% 以上是
  带数字的短行，被误判成索引，导致整章被排除在翻译之外。索引判定现在还要求页面**缺少真正的散文**
  ——出现三段及以上、且不是密集页码清单的长段落，就说明它是章节而非索引。

## 0.2.0

大幅改进**文字版 PDF 的抽取**，源于一个硬骨头（Linus Torvalds《Just for Fun》）——它的文本层把
干净正文和被逐字打散的边栏混在一起。

- **自动回退到 `pdftotext -raw`**：有些 PDF 把边栏/方框文字排成了默认模式会逐字打散的字体
  （`a m i c rokerne l`）。抽取时现在会测量"孤立单字母"的比例，发现默认输出被打散就切到 `-raw`
  （按内容流顺序），救回此前完全无法翻译的乱码页。
- **软连字符去断词**：两端对齐换行处残留的软连字符（U+00AD）会被去掉并把词接回（`win­ ter` →
  `winter`）；在典型的两端对齐排版里，约每 5 段就有 1 段受影响。
- **数字去空格**：被松散字距打散的数字（`1 9 1 7`、`3 5 0 ,000`）会重新拼合；只合并严格夹在数字
  之间的空格，正文和小数不受影响。
- **裁掉尾部索引/封底**：没有标题可切分、密布页码的索引（以及其后的腰封文案、定价、条码乱码）
  现在会从末尾裁掉。
- **更严格的 PDF 标题识别**：单字母、说话人标记（`L:`）、纯罗马数字（`IV.`）、孤立的全大写技术词
  （`LINUX`）不再算作章节标题，对话体、全大写偏多的回忆录不会再被切成几十个假章节。
- **基于内容的前后页跳过现在也覆盖 PDF**：0.1.1 为 EPUB 加的索引/版权页识别，现在按 section 应用到
  PDF 前端。`--no-auto-skip` 可关闭。

## 0.1.2

- **修复长章节被误判为版权页而跳过**：基于内容的前置/后置页识别此前只要在文档开头几段发现版权页
  常见措辞（如 "printed in …"）就判定为前后页，于是正文只是顺带提到某内容"印在"（printed in）某
  报纸上的长篇叙事章节（如《The Power Broker》第 42 章，约 1.1 万词）被错判成版权页、自动跳过未译。
  正文样板措辞检查现在加了长度门槛，只对短文档（< 300 词）生效——真正的版权页/版本说明页本就这么短。

## 0.1.1

- **输出文件名改为纯 ASCII**：生成的文件现在是 `<原名> - Bilingual EN-ZH.epub`（加
  `--single-translate` 则为 `<原名> - ZH.epub`），不再用 `中英对照` 后缀；EPUB 的 `dc:title`
  也用同样的英文标签。`--single-translate` 模式现在有独立的 `ZH` 标签，不再被错标成「双语」。
- **内容识别前置/后置页**：发现阶段除了看文件名，还会分析每个文档的内容，自动跳过索引、版权页、
  参考书目、致谢、目录等（即使文件名是 Z-Library 那种通用的 `part00XX.html` 也能认出）。修复了
  1000+ 条目的索引被当作正文翻译的问题。`--no-auto-skip` 可关闭。

## 0.1.0

首次公开发布。

- **EPUB → 双语 EPUB**：基于 spine 的内容发现（任何 EPUB，无需逐书改代码）；多标签翻译（`<p>`、
  `<h1>`–`<h6>`、`<li>`、`<blockquote>`，`--tags` 可配）；跳过 `<sup>`/`<code>`；英文零篡改——
  在每个元素后追加一个带样式的中文兄弟节点。
- **文字版 PDF → 双语 EPUB**：`pdftotext` 抽取 + 段落重建（按行宽判段末、跨页合并、去页眉页脚/页码）；
  从零构建合规 EPUB。扫描件直接拒绝（不含 OCR）。
- **后端**：本机 Claude Code CLI（`claude -p`，走 Claude 订阅——无需 API key）。
- **质量**：自动生成专名术语表；三层质检（确定性检查 → 语义回查 → 自修复）对抗幻觉与漏译。
- **健壮性**：状态全在 SQLite——幂等可续（随时杀掉重跑）；每本书在 `runs/<slug>/` 下隔离。
- **参数**：`--tags`、`--translation-style`、`--single-translate`、`--skip`、`--min-words`、
  `--concurrency`、`--qa-sample`、`--test-file` 等。
