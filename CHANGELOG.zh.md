# 更新日志

[English](CHANGELOG.md) | **简体中文**

## 0.2.5

- **抽取失败时响亮中止,而不是生成空书** —— 当发现阶段找不到可翻译段落(全部为空、低于
  `--min-words`、被 `--skip` 命中,或被识别为前后页废料)时,`extract` 现在会带着点明各个开关的诊断
  中止(退出码 1),而不再打印 `✓ extract done … 0` 后继续产出一本空的双语 EPUB。这种"假成功"正是
  0.2.3 的 `index_split` bug 一直隐形的原因。`discover_targets` 里的解析失败现在也会报出来(哪个
  文件、什么错误),不再被静默跳过。

## 0.2.4

- **`run --pdf` 像 `run --epub` 一样触发重抽取**：给 `run` 传源文件现在两种格式都会（重新）抽取；
  此前只有 `--epub` 会，于是 `run --pdf book.pdf` 会悄悄沿用旧的 units。重抽取是幂等的（同 PDF →
  同段落哈希 → 全部命中缓存），续跑依然很便宜。
- **回退到上次活动书时给出警告**：不带 `--book`/`--epub`/`--pdf` 的命令会用 `runs/active.txt` 里的
  slug，而并发运行会互相改写它；现在会打印一行警告，避免多书会话里悄悄操作错书。要明确就传 `--book`。

## 0.2.3

- **Calibre / Z-Library 导出的 EPUB 不再抽出零章** —— 这类工具把整本书拆成名为
  `index_split_<n>.html` 的逐章文件,而默认 `--skip` 列表里的 `index` token 以子串方式命中了其中
  每一个,于是整本书被过滤掉(`0 content docs`)。移除了 `index` 这个文件名 token;真正的索引页仍由
  基于内容的前后页检测(`looks_like_matter`)拦截。一本典型的 Z-Library 长篇(《The Mountain in
  the Sea》)现在能抽出 54 章(其中 1 个真索引页按内容自动跳过),而不再是 0 章。

## 0.2.2

- **页码变化也能剥掉的页眉**：页眉以"短语 + 页码"形式重复出现，页码每页都变，所以旧的"整行完全相同"
  频率检测从来没认出它；在那些因边栏而走 `-raw` 抽取的书里，它漏进了约六分之一的段落，有时还插进词
  中间（`the teleLinus Torvalds and David Diamond xix phone`）。现在计数前先把页码归一化，并在页面
  任意位置剥除页眉行（不只页眉页脚），顺带让跨页断开的词重新接上（`tele-`/`phone` → `telephone`）。
  页码按规范罗马数字校验，所以由罗马字母组成的英文词（`did`、`mill`）不会被误当页码。
- **重抽取时清理孤儿译文**：重抽取改变段落文本后（例如清洗修复），旧文本的译文不再滞留缓存；只保留被
  当前段落引用的行，QA 和报告不再处理永远不会出现在成品里的文本。

## 0.2.1

- **修复：日期密集的短章节不再被当成索引跳过**：基于内容的前后页废料检测里，有一条会把"几乎全是
  短条目、且大多带页码样数字"的页面判定为索引/目录。一篇*短*的叙事章节，若塞满笔记本引文、日期和
  脚注编号（如 *Leonardo da Vinci* 第 19 章「Personal Turmoil」，约 1700 词），就会有 60% 以上是
  带数字的短行，被误判成索引，导致整章被排除在翻译之外。索引判定现在还要求页面**缺少真正的散文**
  ——出现三段及以上、且不是密集页码清单的长段落，就说明它是章节而非索引。

## 0.2.0

大幅改进**文字版 PDF 的抽取**，源于一个硬骨头（Linus Torvalds《Just for Fun》）——它的文本层把
干净正文和被逐字打散的边栏混在一起。

- **自动回退到 `pdftotext -raw`**：有些 PDF 把边栏/方框文字排成了默认模式会逐字打散的字体
  （`a m i c rokerne l`）。抽取时现在会测量"孤立单字母"的比例，发现默认输出被打散就切到 `-raw`
  （按内容流顺序），救回此前完全无法翻译的乱码页。
- **软连字符去断词**：两端对齐换行处残留的软连字符（U+00AD）会被去掉并把词接回（`win­ ter` →
  `winter`）；在典型的两端对齐排版里，约每 5 段就有 1 段受影响。
- **数字去空格**：被松散字距打散的数字（`1 9 1 7`、`3 5 0 ,000`）会重新拼合；只合并严格夹在数字
  之间的空格，正文和小数不受影响。
- **裁掉尾部索引/封底**：没有标题可切分、密布页码的索引（以及其后的腰封文案、定价、条码乱码）
  现在会从末尾裁掉。
- **更严格的 PDF 标题识别**：单字母、说话人标记（`L:`）、纯罗马数字（`IV.`）、孤立的全大写技术词
  （`LINUX`）不再算作章节标题，对话体、全大写偏多的回忆录不会再被切成几十个假章节。
- **基于内容的前后页跳过现在也覆盖 PDF**：0.1.1 为 EPUB 加的索引/版权页识别，现在按 section 应用到
  PDF 前端。`--no-auto-skip` 可关闭。

## 0.1.2

- **修复长章节被误判为版权页而跳过**：基于内容的前置/后置页识别此前只要在文档开头几段发现版权页
  常见措辞（如 "printed in …"）就判定为前后页，于是正文只是顺带提到某内容"印在"（printed in）某
  报纸上的长篇叙事章节（如《The Power Broker》第 42 章，约 1.1 万词）被错判成版权页、自动跳过未译。
  正文样板措辞检查现在加了长度门槛，只对短文档（< 300 词）生效——真正的版权页/版本说明页本就这么短。

## 0.1.1

- **输出文件名改为纯 ASCII**：生成的文件现在是 `<原名> - Bilingual EN-ZH.epub`（加
  `--single-translate` 则为 `<原名> - ZH.epub`），不再用 `中英对照` 后缀；EPUB 的 `dc:title`
  也用同样的英文标签。`--single-translate` 模式现在有独立的 `ZH` 标签，不再被错标成「双语」。
- **内容识别前置/后置页**：发现阶段除了看文件名，还会分析每个文档的内容，自动跳过索引、版权页、
  参考书目、致谢、目录等（即使文件名是 Z-Library 那种通用的 `part00XX.html` 也能认出）。修复了
  1000+ 条目的索引被当作正文翻译的问题。`--no-auto-skip` 可关闭。

## 0.1.0

首次公开发布。

- **EPUB → 双语 EPUB**：基于 spine 的内容发现（任何 EPUB，无需逐书改代码）；多标签翻译（`<p>`、
  `<h1>`–`<h6>`、`<li>`、`<blockquote>`，`--tags` 可配）；跳过 `<sup>`/`<code>`；英文零篡改——
  在每个元素后追加一个带样式的中文兄弟节点。
- **文字版 PDF → 双语 EPUB**：`pdftotext` 抽取 + 段落重建（按行宽判段末、跨页合并、去页眉页脚/页码）；
  从零构建合规 EPUB。扫描件直接拒绝（不含 OCR）。
- **后端**：本机 Claude Code CLI（`claude -p`，走 Claude 订阅——无需 API key）。
- **质量**：自动生成专名术语表；三层质检（确定性检查 → 语义回查 → 自修复）对抗幻觉与漏译。
- **健壮性**：状态全在 SQLite——幂等可续（随时杀掉重跑）；每本书在 `runs/<slug>/` 下隔离。
- **参数**：`--tags`、`--translation-style`、`--single-translate`、`--skip`、`--min-words`、
  `--concurrency`、`--qa-sample`、`--test-file` 等。
